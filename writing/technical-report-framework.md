# 技术报告写作框架 —— 从景宁生态流量方案对比中总结

> 来源：2026-03-11 ZTL版 vs DH版 对比分析
> 目的：建立自己的技术报告写作思维框架

## 核心认知转变

**旧思维**：把该写的内容写完 → 任务完成
**新思维**：在回答审查者可能问的所有问题 → 建立说服力

技术报告不是信息的罗列，而是一场**有组织的论证**。

## 五维框架 TLEAP

| 维度 | 英文 | 中文 | 一句话 |
|------|------|------|--------|
| T | Trend | 趋势性 | 不仅说现状，还要说变化 |
| L | Logic | 逻辑性 | 先问题后方案，先诊断后药方 |
| E | Evidence | 证据链 | 每个结论都有"因为→所以" |
| A | Accuracy | 精确度 | 每个词都用对，每个数都有出处 |
| P | Prediction | 预判力 | 提前回答审查者会问的问题 |

## T - 趋势性

### 我犯的错
2.1.3节只写了2024年供水量和用水量，没分析2020-2024的变化趋势。

### 正确做法
有时间序列数据 → 必须做趋势分析：
1. 列出多年数据表
2. 指出总量变化方向和幅度
3. 指出结构性变化（哪个涨了哪个降了）
4. 解读变化的含义

### 检查问句
- "我列的数据跨越多年吗？如果是，我分析趋势了吗？"
- "读者看完我的数据，能判断未来走向吗？"

## L - 逻辑性

### 我犯的错
第4章先列"泄放设施改造方式"（通用方法），再介绍水库现状。读者还不知道水库长什么样就看到了改造方案。

### 正确做法
**问题驱动三步法**：
```
Step 1: 现状（问题在哪） → 读者建立认知
Step 2: 方法（怎么解决） → 读者理解路径
Step 3: 方案（具体怎么做） → 读者做判断
```

### 常见逻辑错误
| 错误模式 | 例子 | 修正 |
|---------|------|------|
| 先方法后问题 | 先列改造方式再介绍水库 | 先介绍水库再列改造方式 |
| 先软件后硬件 | 先说监测设施再说泄放设施 | 先泄放后监测 |
| 结论在前，论据在后 | 开头就说"分析对象为2座" | 先排查完再得出结论 |

### 检查问句
- "读者看到这一节时，前面的信息够他理解吗？"
- "我是在展示知识，还是在解决问题？"

## E - 证据链

### 我犯的错
1. 说"38座已完成核定"但没说在哪个工作任务中完成的
2. 3.2.2说"无珍稀物种"但前文没有水环境评价支撑

### 正确做法
**每个结论的论证模板**：
```
[引用政策/数据] → [分析推导] → [得出结论]

例：根据《浙江省水利厅关于开展已建大中型水库生态流量核定与保障工作的通知》
（政策依据），3座中型水库和38座小型水库分别在大中型水库生态流量核定和
小水电生态流量核定的工作任务中完成（工作归口），目前剩余3座小型水库
未开展生态流量核定工作（结论）。
```

### 检查问句
- "如果审查者在这里画个圈写'依据？'，我能回答吗？"
- "这个定性判断的数据支撑在前文哪一段？"

## A - 精确度

### 我犯的错
1. "本次…本次…"两次开头
2. "尚未核定"和"未开展"混用（含义不同）
3. 用"考虑到…所以采用"替代直接说明技术规则

### 对照表
| 模糊 | 精确 |
|------|------|
| 考虑到资料系列较长 | 资料系列n=62年，满足SL/T 712规范要求（n≥30年） |
| 对下游影响较小 | 集雨面积0.31km²，不足0.5km²，按省厅要求不纳入 |
| 征求有关部门意见 | 根据水资源条件和开发利用状况，选择合适的计算方法 |
| 均已完成 | 分别在大中型水库核定和小水电核定工作中完成 |

### 检查问句
- "同一个意思我用了几个不同的词？该统一吗？"
- "能用数字替代形容词吗？"

## P - 预判力

### 我犯的错
没有站在审查者的角度审视全文。DH版增加了水环境评价和趋势分析，正是因为预判到审查者会问。

### 审查者常问的问题模板
| 审查者会问 | 你需要写什么 |
|-----------|-------------|
| 政策依据是什么？ | 每项工作引用具体文件和文号 |
| 下游生态环境如何？ | 水环境功能区达标率、水质类别 |
| 数据可靠吗？ | 数据来源、系列长度、参证站信息 |
| 现有设施能改吗？ | 先写设施现状再写改造方案 |
| 方法选得对吗？ | 说明方法适用条件和选择理由 |
| 不做有什么后果？ | 说明监管盲区、不达标风险 |

### 检查问句
- "如果我是审查专家，看到这一页会提什么问题？"
- "我是在写给完成任务，还是在写给说服人？"

## 实践路线

### 阶段1：写前列大纲时
- 对照 TLEAP 五个维度，检查每一章是否覆盖

### 阶段2：写作过程中
- 每完成一节，问自己那个维度的"检查问句"

### 阶段3：完稿自查
- 请 CC 用 report-writing skill 做全文审查

### 阶段4：对比学习
- 拿到别人写的好报告，用 TLEAP 分析它好在哪里
- 拿到被退回的报告，用 TLEAP 分析它差在哪里
